## What Was Built

### Phase 0: RAG Foundation Upgrade
The search system was upgraded from simple vector-only search to a hybrid architecture.

**What changed:**
- **Contextual chunk embeddings** (`services/embeddings.py`) — when embedding transcript chunks, meeting metadata (title, date, participants) is now prepended to each chunk before generating the embedding vector. The stored text stays raw for display, but the embedding captures full context.
- **Full-text search** (`scripts/setup_supabase.sql`) — added PostgreSQL `tsvector` columns with GIN indexes on the `embeddings` and `decisions` tables, plus a new `search_embeddings_fulltext()` RPC function for keyword-based search.
- **Hybrid search with RRF** (`services/supabase_client.py`) — `search_memory()` now runs both vector (semantic) and full-text (keyword) searches, then merges results using Reciprocal Rank Fusion. This means searching for "AWS" will find exact keyword matches AND semantically related discussions about cloud providers.
- **Cross-reference enrichment** (`services/supabase_client.py`) — search results now come enriched with the meeting title, related decisions, and tasks from the same meeting.
- **Updated query prompt** (`core/system_prompt.py`) — Claude now receives the enriched context when answering questions.

**Why it matters:** Before this, searching "cloud provider" might miss a discussion about "AWS vs Azure" because the words don't overlap. Now both semantic meaning AND exact keywords are searched, and results include the full meeting context.

**New tests:** 27 in `tests/test_rag_search.py`

### Phase 1: Meeting Prep Documents
Gianluigi now auto-generates preparation documents before upcoming meetings.

**What changed:**
- **Implemented all stub functions** in `processors/meeting_prep.py`:
  - `find_related_meetings()` — searches past meetings related to the upcoming topic
  - `find_relevant_decisions()` — finds decisions relevant to the meeting agenda
  - `find_participant_tasks()` — looks up open tasks for each participant
  - `get_stakeholder_context()` — pulls info from the Stakeholder Tracker
  - `format_prep_document()` — generates a clean Markdown document
- **Fixed broken scheduler** (`schedulers/meeting_prep_scheduler.py`) — was calling a non-existent `embedding_service.search_similar()` method; now uses the working search pattern.
- **Sensitivity-aware distribution** — sensitive meeting preps go only to Eyal; normal ones go to the full team.

**New tests:** 19 in `tests/test_meeting_prep.py`

### Phase 2: Weekly Digest
Gianluigi auto-generates a weekly summary every Sunday evening.

**What changed:**
- **Implemented all stub functions** in `processors/weekly_digest.py`:
  - `get_meetings_for_week()` — queries meetings in the date range
  - `get_decisions_for_week()` — decisions with meeting titles
  - `get_task_summary()` — categorizes tasks: completed, overdue, due next week
  - `get_open_questions_summary()` — all unresolved questions
  - `get_upcoming_meetings()` — next 7 days with CropSight filter
  - `format_digest_document()` — Markdown weekly report
- **New scheduler** (`schedulers/weekly_digest_scheduler.py`) — runs hourly, fires Sunday 18:00-20:00. Has `generate_now()` for manual trigger. Tracks `_last_digest_week` to prevent duplicates.
- **Wired into main.py** — starts/stops with other schedulers.
- **Saves to Google Drive** in the Weekly Digests folder, then sends via email + Telegram.

**New tests:** 30 in `tests/test_weekly_digest.py`

### Phase 3: Gmail Read (Email Inbox Processing)
Gianluigi can now read and process incoming emails.

**What changed:**
- **Added `gmail.modify` scope** to `services/gmail.py` — needed for marking emails as read.
- **New email watcher** (`schedulers/email_watcher.py`) — polls inbox every 5 minutes, routes team emails:
  - **Questions** -> Claude agent generates a reply -> sends response via email
  - **Attachments** -> document ingestion pipeline (summarize, embed, store)
  - **Approval replies** -> approval flow (APPROVE/REJECT in subject line)
- **Wired into main.py** — starts alongside other watchers.

**Action required:** Eyal must re-run `scripts/get_google_token.py` to get the new `gmail.modify` scope in the OAuth token.

**New tests:** 29 in `tests/test_email_watcher.py`

When meetings mention new contacts or organizations, Gianluigi suggests updates to the Stakeholder Tracker.

- **Google Sheets update** (`services/google_sheets.py`) — `apply_stakeholder_update()` either updates an existing row or appends a new one.

**New tests:** 12 in `tests/test_stakeholder_updates.py`

### Phase 5: Pre-Meeting Reminders
Gianluigi sends a brief context reminder 2-3 hours before meetings.

**What changed:**
- **Added to meeting prep scheduler** (`schedulers/meeting_prep_scheduler.py`):
  - `_send_pre_meeting_reminder()` — fires 2-3 hours before CropSight meetings

  - Tracks `_reminders_sent` set to avoid duplicates

**New tests:** 5 added to `tests/test_meeting_prep.py`

### Phase 6: Auto-Publish with 30-Minute Review Window
Optional mode where summaries auto-publish after a review window.

**What changed:**
- **New settings** (`config/settings.py`):
  - `APPROVAL_MODE` — `"manual"` (default, current behavior) or `"auto_review"`
  - `AUTO_REVIEW_WINDOW_MINUTES` — default 30
- **Timed auto-publish** (`guardrails/approval_flow.py`) — in `auto_review` mode, an asyncio background task auto-approves after N minutes. If Eyal acts before the timer, the pending auto-publish is cancelled.
- **`/retract` command** (`services/telegram_bot.py`) — undoes the last auto-published summary.
- **Countdown indicator** — approval messages show "Auto-publishes in 30 min" in `auto_review` mode.

**New tests:** 14 in `tests/test_auto_publish.py`
