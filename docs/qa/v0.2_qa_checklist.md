## Phase 0: RAG / Hybrid Search

### Test: Hybrid search returns better results than before
**How:** Start the bot (`python main.py`), then send a Telegram message asking about a topic from an existing meeting.

- [ ] Ask: "What did we decide about cloud providers?" (or similar topic from your MVP Focus transcript)
- [ ] Verify the response includes relevant transcript excerpts
- [ ] Verify the response mentions the meeting title and date (enrichment working)
- [ ] Verify related decisions are referenced in the answer

### Test: Keyword search works
- [ ] Ask about something with a specific keyword that wouldn't match semantically (e.g., a person's name, a company name, an acronym)
- [ ] Verify it finds matches even when the semantic meaning is unrelated

### Test: Search with no results
- [ ] Ask about a topic that has never been discussed
- [ ] Verify Gianluigi responds gracefully (not an error)

## Phase 1: Meeting Prep Documents

### Test: Manual meeting prep trigger
**How:** Find an upcoming CropSight event on Google Calendar, note its event ID.

- [ ] Send via Telegram: "Prepare me for the meeting with [topic/name]"
- [ ] Verify Gianluigi generates a prep document
- [ ] Verify it includes: related past meetings, relevant decisions, participant tasks, stakeholder context
- [ ] Verify the document is saved to Google Drive (Meeting Prep folder)
- [ ] Check: if the meeting is sensitive, only Eyal receives it; if normal, the team gets it

### Test: Automatic meeting prep (scheduler)
- [ ] Verify the meeting prep scheduler is running (check logs after `python main.py`)
- [ ] Wait for a meeting that's within the 24-hour prep window
- [ ] Verify a prep doc is auto-generated and sent
- [ ] Verify a second run does NOT generate a duplicate prep for the same meeting

### Test: Pre-meeting reminder
- [ ] With a CropSight meeting 2-3 hours away, verify a Telegram reminder is sent
- [ ] Verify the reminder includes brief context and open task count
- [ ] Verify the same meeting doesn't trigger a second reminder

## Phase 2: Weekly Digest

- [ ] Verify it includes sections: Meetings, Decisions, Task Summary, Open Questions, Upcoming Meetings
- [ ] Verify it's saved to Google Drive (Weekly Digests folder)
- [ ] Verify it's sent via email to the team
- [ ] Verify it's posted to the Telegram group

### Test: Digest with no activity
- [ ] Trigger a digest for a week with no meetings (e.g., `generate_weekly_digest` with a far-past `week_start`)
- [ ] Verify it generates cleanly with "No meetings this week" messages (not errors)

### Test: Automatic Sunday trigger
- [ ] On a Sunday between 18:00-20:00, verify the scheduler fires
- [ ] Verify it doesn't fire again for the same week
- [ ] Verify it doesn't fire on other days

## Edge Cases to Verify

- [ ] **No internet / API down** — start bot with invalid API key, verify graceful error messages (not crashes)
- [ ] **Empty transcript** — upload a file with minimal/no content, verify no crash
- [ ] **Very long transcript** — verify chunking works for a 2+ hour meeting transcript
- [ ] **Concurrent processing** — upload 2 transcripts at once, verify both are processed without conflicts
- [ ] **Bot restart** — stop and restart `python main.py`, verify all services reconnect cleanly
- [ ] **Supabase connection loss** — verify the bot doesn't crash if Supabase is temporarily unreachable

## Performance Checks

- [ ] **Search response time** — Telegram question should get a response within ~10 seconds
- [ ] **Transcript processing time** — a 1-hour transcript should process in under 2 minutes
- [ ] **Memory usage** — monitor RAM usage over 1+ hour of running (should stay stable)
- [ ] **No polling storms** — verify schedulers respect their intervals (15 min transcript watcher, 5 min email watcher, etc.)
