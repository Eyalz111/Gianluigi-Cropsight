## Phase 2: Outline Proposal (Core Flow)

### O1. Receive Outline Proposal
- [ ] Have a CropSight meeting scheduled 12-24 hours from now
- [ ] Wait for scheduler to detect it (or trigger manually)
- [ ] Receive outline proposal on Telegram
- [ ] Verify outline contains:
  - Meeting title and time
  - Participant list
  - Data sections (tasks, decisions, etc.) — some may show "unavailable" if no data
  - Suggested agenda
- [ ] Verify inline buttons appear:
  - If confident match: `[Generate as-is]` `[Add focus]` `[Skip this prep]`

- [ ] For a known recurring meeting (e.g., "Tech Review" with Roye on Tuesday):
  - Verify it's classified as `founders_technical` with high confidence
  - Verify buttons do NOT include "Wrong type?"
- [ ] For an unknown/new meeting:
  - Verify it falls back to `generic` template
  - Or shows "ask" confidence with "Wrong type?" button

### O3. Graceful Degradation
- [ ] Check outline sections — if some data queries failed (e.g., no tasks for a participant), verify those sections show "unavailable" rather than crashing
- [ ] The outline should still be complete and usable even with partial data

## Phase 3: Outline Interactions

### I1. "Generate as-is"
- [ ] Tap `[Generate as-is]` on an outline
- [ ] Message updates to "Generating prep document..."
- [ ] Receive a new approval message with the full meeting prep document
- [ ] Verify the prep contains structured sections matching the template
- [ ] Prep awaits your approval (standard approve/edit/reject flow)

### I2. "Add Focus"
- [ ] Tap `[Add focus]` on a NEW outline (or wait for another meeting)
- [ ] Message updates asking: "What should I focus on?"
- [ ] Type a focus instruction, e.g.: "Focus on MVP timeline and what Roye committed to last week"
- [ ] Receive updated outline reflecting your focus
- [ ] New buttons appear: `[Generate]` `[Edit more]` `[Skip]`
- [ ] Tap `[Generate]` — verify the generated prep incorporates your focus instruction

### I3. "Wrong Meeting Type" (if available)
- [ ] On an outline with `[Wrong type?]` button, tap it
- [ ] Receive template picker with options: Founders Technical, Founders Business, Monthly Strategic, Generic
- [ ] Select a different type
- [ ] Receive regenerated outline with the new template's sections
- [ ] Verify the correction is remembered (next time same meeting title should match correctly)

### I4. "Skip"
- [ ] Tap `[Skip this prep]` on an outline
- [ ] Message updates to "Prep skipped for [title]."
- [ ] Verify no further reminders for this meeting
- [ ] Verify no prep document is generated

- [ ] With a meeting 6-12h away, verify outline arrives
- [ ] Check logs for `timeline_mode: urgent`

- [ ] Check logs for "too late" or "skip" message

## Test Execution Tips

**Order:** Run phases 1-4 first (they cover the core happy path). Phases 5-10 test edge cases and can be done selectively.

**Creating test meetings:** Add CropSight meetings to your Google Calendar with:
- Purple color (color 3)
- Invite Roye/Paolo for specific template matching
- Use titles like "Tech Review", "Business Sync", "Board Meeting" for template matching tests

**Triggering the scheduler early:** Instead of waiting for the hourly cycle, you can:
1. Temporarily reduce the scheduler interval in settings
2. Or restart the bot (it runs the check on startup)

**Checking logs:** Use `gcloud run logs read` or the Cloud Run console to monitor real-time logs for scheduler activity, classification results, and timer management.

**Quick smoke test (15 min):** If short on time, just do S1, S3, O1, I1, D1 — this covers the core path from detection to distribution.
