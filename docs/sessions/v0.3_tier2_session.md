## Work Completed (Chronological)

### 1. Entity Extraction Quality Improvements

**Problem:** Initial entity extraction was producing junk (AWS, Google, IBM, Canada, "Grant process opportunity"). Hardcoded blocklist approach wasn't scalable.

**Solution (3 iterations):**
1. Started with hardcoded blocklist — worked but not scalable
2. Replaced with two-pass LLM validation: Haiku extracts → Haiku validates
3. Piggybacked on existing Opus transcript call — zero extra LLM cost, 100% transcript coverage

**Key decisions:**
- Entity extraction prompt uses CropSight-specific stakeholder definition
- Stakeholder schema enriched with `type`, `relationship`, `speaker` fields
- 16K transcript sampling (beginning + middle + end) for fallback Haiku extraction
- `_filter_known_names()` removes team members and participants before DB storage

### 2. Weekly Entity Health Check

Added `review_entity_health()` to `entity_extraction.py`:
- Auto-cleans entities matching team member names
- Flags orphans (entities with no mentions)
- Lists entities created this week
- Integrated into weekly digest as "Entity Registry Health" section

### 3. Settings Centralization

Moved ~25 hardcoded values to `config/settings.py`:
- Search thresholds, chunk sizes, recency half-life
- Alert thresholds (overdue cluster min, stale commitment days, etc.)
- Scheduler intervals (email, transcript, document, meeting prep, digest, reminders, alerts)
- Conversation memory (max messages, TTL)

**Bug found and fixed:** All schedulers had `DEFAULT_X = settings.Y` at module level — captured at import time, making tests unmockable. Fixed by removing module-level constants and resolving defaults at runtime in `__init__`.

### 4. Opus Piggyback

Instead of a separate Haiku call for entity extraction:
- Enriched the existing Opus transcript extraction schema to include stakeholders
- Added `STAKEHOLDER EXTRACTION RULES` to the system prompt
- `extract_and_link_entities()` now accepts `pre_extracted` parameter
- Transcript processor passes `extracted.get("stakeholders", [])` to entity linking
- Result: 100% transcript coverage at zero extra LLM cost
